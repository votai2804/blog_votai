Dependency Injection & IoC Container Trong ASP.NET Core

10 Dec 2024 .NET 16 phút đọc
← Quay lại Blog

Dependency Injection Là Gì?

Dependency Injection (DI) là một design pattern giúp code loose coupling. Thay vì một class tạo dependencies của nó, DI container cung cấp chúng.

Tại Sao Cần DI?

  • Loose Coupling: Classes không phụ thuộc trực tiếp vào implementations
  • Testability: Dễ dàng mock dependencies cho unit tests
  • Flexibility: Dễ dàng thay đổi implementations
  • Maintainability: Code dễ hiểu và dễ bảo trì

Ví Dụ Không Sử Dụng DI

public class OrderService { private readonly EmailService _emailService; public OrderService() { // Tightly coupled _emailService = new EmailService(); } public void CreateOrder(Order order) { // Create order... _emailService.SendEmail(order.CustomerEmail, "Order confirmed"); } }

Sử Dụng Constructor Injection

public interface IEmailService { void SendEmail(string email, string message); } public class EmailService : IEmailService { public void SendEmail(string email, string message) { // Send email logic } } public class OrderService { private readonly IEmailService _emailService; // Dependency được inject qua constructor public OrderService(IEmailService emailService) { _emailService = emailService; } public void CreateOrder(Order order) { // Create order... _emailService.SendEmail(order.CustomerEmail, "Order confirmed"); } }

Cấu Hình DI Container

ASP.NET Core có built-in DI container. Cấu hình trong Program.cs:

builder.Services.AddScoped<IEmailService, EmailService>(); builder.Services.AddScoped<IOrderService, OrderService>(); builder.Services.AddTransient<ILogger, ConsoleLogger>(); builder.Services.AddSingleton<IConfigService, ConfigService>();

Lifetime Management

Transient

Được tạo mới mỗi lần được request:

builder.Services.AddTransient<IService, Service>();

Scoped

Được tạo một lần per request:

builder.Services.AddScoped<IService, Service>();

Singleton

Được tạo một lần và reused cho toàn bộ application:

builder.Services.AddSingleton<IService, Service>();

Property & Method Injection

// Property Injection public class MyClass { public IService Service { get; set; } } // Method Injection public class MyController { public IActionResult MyAction([FromServices] IService service) { // Sử dụng service return Ok(); } }

Repository Pattern + DI

builder.Services.AddScoped<IProductRepository, ProductRepository>(); builder.Services.AddScoped<IProductService, ProductService>(); public class ProductService { private readonly IProductRepository _repository; public ProductService(IProductRepository repository) { _repository = repository; } public async Task<Product> GetProductAsync(int id) { return await _repository.GetByIdAsync(id); } }

Best Practices

  • Luôn sử dụng interfaces, không concrete types
  • Chọn lifetime phù hợp cho mỗi service
  • Tránh circular dependencies
  • Sử dụng constructor injection (ưu tiên)
  • Đừng share mutable state với singletons
  • Test services bằng cách mock dependencies

Kết Luận

Dependency Injection là một pattern cực kỳ quan trọng trong modern software development. Nó giúp viết code cleaner, testable hơn và dễ maintain.
