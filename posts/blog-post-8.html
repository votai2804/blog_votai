Git Flow & Version Control: Quản Lý Mã Nguồn Chuyên Nghiệp

02 Dec 2024 DevOps 17 phút đọc
← Quay lại Blog

Git Là Gì?

Git là distributed version control system phổ biến nhất. Nó giúp quản lý source code changes, collaborate với team, và track project history.

Cơ Bản Git

// Khởi tạo repository git init // Clone repository git clone https://github.com/user/repo.git // Check status git status // Add files to staging area git add . git add filename.txt // Commit changes git commit -m "Add new feature" // Push to remote git push origin main // Pull từ remote git pull origin main

Git Flow Workflow

Git Flow là branching model giúp organize development process:

// Main branches: - main (production-ready code) - develop (integration branch) // Support branches: - feature/* (new features) - release/* (release preparation) - hotfix/* (production bugs)

Feature Development

// Tạo feature branch từ develop git checkout -b feature/user-authentication develop // Làm việc trên feature git add . git commit -m "Implement user login" git push origin feature/user-authentication // Tạo Pull Request trên GitHub // Review và merge vào develop // Xóa feature branch git branch -d feature/user-authentication

Release Management

// Tạo release branch git checkout -b release/1.0.0 develop // Bug fixes trong release git commit -m "Fix bugs for v1.0.0" // Merge vào main git checkout main git merge --no-ff release/1.0.0 git tag -a v1.0.0 -m "Version 1.0.0" // Merge back vào develop git checkout develop git merge --no-ff release/1.0.0 // Xóa release branch git branch -d release/1.0.0

Hotfix Management

Khi cần fix bug trong production:

// Tạo hotfix branch từ main git checkout -b hotfix/critical-bug main // Fix bug git commit -m "Fix critical bug" // Merge vào main git checkout main git merge --no-ff hotfix/critical-bug git tag -a v1.0.1 -m "Version 1.0.1" // Merge vào develop git checkout develop git merge --no-ff hotfix/critical-bug // Xóa hotfix branch git branch -d hotfix/critical-bug

Commit Conventions

Sử dụng conventional commits để code history sạch sẽ:

feat: Thêm feature mới fix: Sửa bug docs: Thay đổi documentation style: Code formatting refactor: Refactor code perf: Performance improvement test: Thêm tests chore: Build hoặc dependency updates Ví dụ: git commit -m "feat: Add user authentication" git commit -m "fix: Fix login timeout issue"

Merge Conflicts

Giải quyết conflicts khi merge:

// Khi có conflict git merge feature/branch // CONFLICT markers sẽ appear // Resolve conflicts manually rồi git add resolved-file.txt git commit -m "Resolve merge conflicts" git push origin main

Rebase vs Merge

Merge

Tạo merge commit, giữ nguyên history:

git merge feature/branch

Rebase

Reapply commits, history sạch hơn:

git rebase main git rebase -i HEAD~3 // Interactive rebase

Best Practices

  • Viết meaningful commit messages
  • Commit thường xuyên, không commit quá lớn
  • Pull before push để avoid conflicts
  • Sử dụng branches cho mỗi feature
  • Code review qua Pull Requests
  • Keep main branch stable
  • Tag versions cho releases

Kết Luận

Git Flow là best practice workflow cho professional software development. Master Git Flow sẽ giúp team organize code, collaborate hiệu quả, và maintain quality.
